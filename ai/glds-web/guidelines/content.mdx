> **Who this is for:** Engineers and content authors writing or reviewing any string on a GL marketing page — headlines, CTAs, error messages, empty states, form labels, or social proof.
> **What this covers:** Great Learning's voice rules, casing, CTA copy patterns, number formatting, hero headline formula, error and empty state rules, and SEO keyword density.

## Voice and tone

Great Learning's voice is **confident, cordial, motivational, human, and concise**.

- **Confident:** sounds assured and credible; never hedges or over-qualifies
- **Human warmth:** speaks directly to the user as a person — "talk to the user as though they are standing right in front of you"
- **Motivational without patronising:** urges action, doesn't overwhelm or condescend
- **Active voice required:** "The course starts 12 July" not "On 12 July the course will begin"
- **Second person preferred:** "your career" not "our learners' careers"
- **No jargon or slang** in main headings or first-fold copy
- **No plagiarism** from other edtech sites

## Casing rules

| Content type | Rule |
|---|---|
| Main headings | Title Case |
| Banner headers | Sentence case (unless full header is a brand/program name → Title Case) |
| Subheadings | Sentence case |
| Body copy | Sentence case |
| CTA buttons | Title Case |
| Navigation items | Title Case |
| Form labels | Title Case |
| Error messages | Sentence case |
| Tooltips | Sentence case |

Always capitalise: course names, university names, product names, proper nouns.

## Hero headline formula

| Element | Rule | Example |
|---|---|---|
| H1 | Title Case, ≤ 6 words, outcome-first | "Master Data Science in 6 Months" |
| Subhead | Sentence case, ≤ 120 characters, specifics + proof | "IIT Roorkee certificate, live mentorship, 95% placement record." |
| CTA | Title Case, ≤ 3 words, action + value | "Enrol Now" / "Start Free Trial" |

Hero copy must survive **40px mobile H1** without an awkward line break — keep it short.

## CTA copy rules

```
✅ "Enrol Now"           — action + value
✅ "Start Free Trial"    — action + value
✅ "Book A Demo"         — action + value
✅ "Get Free Brochure"   — action + value
✅ "Log In"              — two words, not "Login"
✅ "Sign Up"             — two words, not "Signup"

❌ "Submit"              — vague
❌ "Click Here"          — vague
❌ "ENROL FOR FREE"      — all-caps not allowed
❌ "Enrol for free"      — not Title Case for a CTA
```

- No full stops or colons in CTA text
- No parentheses in CTA text
- Never write "etc.", "and more", or "among others" in CTAs or headings — they lose clarity

## Number and social proof formatting

```
✅ 10K learners          — abbreviate at 10,000+
✅ 4.8/5 rating          — show the scale
✅ 95% placement record  — state the metric
✅ ₹ not INR, Rs, or Rupees
✅ 4:00 PM–5:00 PM IST   — en dash for ranges, always include timezone
✅ 12 May 2025           — international date format, no superscript
✅ 12th May 2025         — domestic (India) format

❌ 10,000 learners       — too long for a stat callout
❌ Rs 25,000             — use ₹
❌ "100s of learners"    — vague; use a real number
```

Social proof copy pattern: **real name, real photo, real outcome**.

```
✅ "8,000 graduates completed the Data Science course in 12 months"
✅ "4.8/5 rated by 12,000+ learners"
✅ "Join 2 million+ learners from 150+ countries"
❌ "Thousands of happy learners" — no number, no credibility
```

## Error message rules

Every error must answer three questions: **What happened? Why? What should the user do next?**

```
✅ "We couldn't load your results. Check your connection and try again."
✅ "There seems to be an issue. Shall we give it another try?"

❌ "Something went wrong."           — what happened? what to do?
❌ "Looks like you made a mistake."  — condescending tone
❌ "Error 404"                       — unexplained, unhelpful
```

Error message text is always Sentence case. No exclamation marks. No "Oops" or "Uh oh" openers on public-web error states.

## Empty state rules

Never leave a section blank without explanation. Empty states must explain what will appear and when.

```
✅ "Your saved courses will appear here once you bookmark a program."
✅ "No results found for 'blockchain'. Try a broader search term."

❌ (blank section with no text)
❌ "No results found." — why? what should I do?
```

## Do / Don't

| Do ✓ | Don't ✗ |
|---|---|
| Title Case for CTAs: "Start Free Trial" | All-caps: "START FREE TRIAL" or lowercase: "Start free trial" |
| Active voice: "The course starts 12 July" | Passive: "The course will be starting on 12 July" |
| Error: What happened + why + next action | Error: "Something went wrong." |
| Numbers: "10K learners", "₹25,000" | Vague: "Thousands of learners", "Rs 25,000" |
| Hero H1 ≤ 6 words | 12-word headline that breaks on 40px mobile |
| Social proof with name + photo + outcome | Anonymous quotes with no attribution |
| "Log In" and "Sign Up" (two words) | "Login" or "Signup" (one word) |

## SEO keyword density (program and course pages)

Apply to all program, course, and landing pages. **Skip for Olympus and internal app UI.**

- Primary keyword density: **1–2% of total page word count** is the ceiling
- Above 2%: flag and suggest semantic variants
- Above 5%: mandatory revision before publish — Google treats this as spam

Required keyword placements (once each, does not count toward density cap):
- Page `<title>` tag
- Meta description
- H1
- First 100 words of body copy
- At least one H2 or H3

After required placements, use semantic variants — not the exact phrase repeated. Example: "Python for Data Science" → "this course", "data analysis with Python", "the program".
