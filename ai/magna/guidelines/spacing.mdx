> **Who this is for:** Engineers laying out screens, cards, and lists with `YStack` / `XStack` / `Surface` from `@gl/elements`.
> **What this covers:** Magna's spacing tokens, why surfaces breathe (vs. Jedi's density), and the 48dp mobile touch-target rule.

## Why it matters

Spacing is how a learner-facing surface signals calm. Tight gaps read as "system" or "admin"; generous gaps read as "course material". Magna is mobile-first — the gap between two cards is the difference between a confident swipe and a mis-tap.

## Core principles

1. **Use the `$` scale, never raw px.** `$xs`, `$sm`, `$md`, `$lg`, `$xl`, `$1`–`$10`. Off-grid values like `11px` are a smell.
2. **Surfaces breathe.** `$md`+ gap between cards, `$lg`+ section padding, `$xl` between page bands.
3. **Touch targets ≥ 48×48dp** — larger than Jedi's 44 because Magna is mobile-first.
4. **Spacing between targets ≥ 8dp** so a thumb doesn't catch two at once.
5. **Use `gap`, not `marginBottom` between siblings.** `YStack`/`XStack` is the right tool.

## How to implement

### Page section
```tsx
import { YStack, XStack, Typography } from '@gl/elements';

<YStack gap="$lg" padding="$xl">
  <Typography variant="h3">Your enrolled courses</Typography>
  <XStack gap="$md" flexWrap="wrap">
    {courses.map((c) => <CourseCard key={c.id} {...c} />)}
  </XStack>
</YStack>
```

### Mobile touch target
```tsx
import { Button } from '@gl/elements';

<Button onPress={enroll} size="large">  {/* ≥ 48dp height */}
  Enroll
</Button>
```

## Do / Don't

| Do | Don't |
|---|---|
| `gap="$md"` between cards | `gap={11}` |
| `padding="$lg"` for section padding | `paddingLeft: '17px'` |
| Use `YStack` / `XStack` with `gap` | Add `marginBottom` between siblings |
| Target ≥ 48×48dp | Ship a 32px tap target on mobile |
