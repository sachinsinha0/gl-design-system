> **Who this is for:** Engineers and designers picking type styles for learner-facing screens in `gl-app-native`.
> **What this covers:** The Inter scale Magna ships, naming conventions for variants, and the rules that diverge from Jedi (mobile body, title-case buttons).

## Why it matters

Learners read in motion — on phones, between classes, sometimes with one hand. Type that shrinks on mobile or hides hierarchy behind weight loses them. Magna picks a closed, predictable scale so devs spend zero cognitive energy on font choices and learners get the same readable surface every screen.

## Core principles

1. **One family, three weights.** Inter at 400 / 500 / 600 — never load 300 or 800.
2. **The variant names are fixed.** `h1`–`h5`, `subtitle1`/`2`, `body1`/`2`, `caption`, `overline`, `buttonLarge`/`Medium`/`Small`. No `h6`, no `headline1`.
3. **Body stays 16/14 on mobile.** Unlike Jedi, Magna does not shrink body text — learner legibility wins over density.
4. **Title-case buttons** (only `overline` is uppercased).
5. **Hierarchy comes from variant + role color** (not arbitrary `fontSize` / `fontWeight` overrides).

## How to implement

### Headings, body, metadata
```tsx
import { Typography } from '@gl/elements';

<Typography variant="h3">Lesson 4 · State management</Typography>
<Typography variant="body1">React state lives in the component that owns it.</Typography>
<Typography variant="caption" color="$onSurfaceVariant">Updated 2 hours ago</Typography>
```

### Button text (title case)
```tsx
import { Button } from '@gl/elements';

<Button onPress={save}>Save changes</Button>      {/* ✅ Title Case */}
<Button onPress={save}>SAVE CHANGES</Button>      {/* ❌ Don't UPPERCASE */}
```

## Do / Don't

| Do | Don't |
|---|---|
| `variant="h3"` for a section heading | `variant="headline3"` (not exported) |
| Title-case button labels | UPPERCASE button labels |
| Use `caption` for metadata | Reach for 13/15/17px — the scale is closed |
| Stick to weights 400/500/600 | Load 300 (Light) or 800 (ExtraBold) |
