> **Who this is for:** Engineers shipping internal / partner / admin UIs — every form, every alert, every interactive control.
> **What this covers:** Rules derived from **measured contrast ratios of the shipped Jedi values** — they're facts of this system, not generic web advice.

## Why it matters

Partner-facing UIs are the only Great Learning surface where regulators and procurement teams ask for WCAG conformance reports. The rules below come from running every shipped color through a contrast checker — `error.main` is 3.64:1 on white, so it **fails AA for text** no matter how official it looks. Following the rule is faster than re-litigating it per ticket.

## Core principles

1. **Colored text in light mode uses `.dark`** — never `.main`.
2. **Warning has no text-safe solid in light mode.** Use `text.primary` plus a warning-colored icon, or the 160p/190p tint recipe.
3. **Secondary (orange) is decorative, not semantic.** Never used to mean caution.
4. **Tinted surfaces:** bg `shades-190-p`, text `shades-160-p`.
5. **Never communicate state through color alone.** Every error has icon + message.
6. **Focus must be visible.** Don't suppress MUI's focus ring.
7. **Touch targets ≥ 44×44px** on partner-facing surfaces.
8. **Disabled = `action.disabled` + `action.disabled-background`**, never `opacity: 0.5`.
9. **Body text below 12px is forbidden** on partner-facing surfaces.
10. **One `<h1>` per page**; descend without skipping heading levels.

## How to implement

### Semantic markup (links navigate, buttons act)
```tsx
import { Link, Button } from '@mui/material';

// ✅ Action that looks like a link
<Link component="button" onClick={showMore}>Show more</Link>

// ✅ Real navigation
<Link href={`/cohorts/${id}`}>View cohort</Link>

// ❌ An <a> with a JS handler that doesn't navigate
<a href="#" onClick={showMore}>Show more</a>
```

### Icon-only controls
```tsx
import { IconButton, Tooltip } from '@mui/material';
import CloseIcon from '@mui/icons-material/Close';

<Tooltip title="Close">
  <IconButton aria-label="Close dialog" onClick={close}>
    <CloseIcon />
  </IconButton>
</Tooltip>
```

### Disabled (use the prop)
```tsx
<Button disabled>Save</Button>                          {/* ✅ */}
<Box sx={{ opacity: 0.5, pointerEvents: 'none' }}>...</Box>  {/* ❌ */}
```

### Warning copy (no text-safe main)
```tsx
import { Alert } from '@mui/material';

<Alert severity="warning">This action affects 42 learners.</Alert>     {/* ✅ tint recipe */}
<Typography color="warning.main">This action affects 42 learners.</Typography>  {/* ❌ 1.65:1 fails */}
```

## Do / Don't

| Do | Don't |
|---|---|
| `<Alert severity="warning">` for caution copy | `<Typography color="warning.main">Caution</Typography>` |
| `<IconButton aria-label="Close">…</IconButton>` | `<IconButton>…</IconButton>` |
| `<Link component="button">Edit</Link>` for inline action | `<a href="#" onClick={edit}>Edit</a>` |
| Use the `disabled` prop | `sx={{ opacity: 0.5, pointerEvents: 'none' }}` |
