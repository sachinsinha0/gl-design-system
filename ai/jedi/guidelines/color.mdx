> **Who this is for:** Engineers building internal tools, partner dashboards, and admin UIs with `@gl/jedi`.
> **What this covers:** The "one loud blue, calm ink" palette — when to use `.main` vs `.dark`, why `secondary` is decorative-only, and the tinted-surface recipe for alerts and banners.

## Why it matters

Staff and partners live in Jedi for hours. Color noise compounds — every red border, every orange chip drains attention. Jedi is built so that one color (Primary blue) carries every interactive moment, ink stays near-black, and feedback always passes contrast. The discipline isn't aesthetic — it's measured: `error.main` only scores 3.64:1 on white, which **fails** AA for text.

## Core principles

1. **Source of truth: `getColors(mode)`.** Never paste a hex into component code.
2. **Status text uses `.dark`** in light mode — `primary.main` (4.96:1) and `error.dark` (5.55:1) pass AA; `error.main` / `success.main` / `warning.main` do not.
3. **Tinted alerts use the 160p / 190p recipe** — `<Alert>` already does this for you.
4. **Secondary (orange) is decorative warmth, not caution.** Caution belongs to `warning` via the tint recipe.
5. **Dividers are sparse (6% by design).** Default to whitespace; reach for `<Divider>` only when spacing + outlined surface won't do.

## How to implement

### Status text
```tsx
import { Typography } from '@mui/material';

<Typography color="error.dark">Could not save changes.</Typography>     {/* ✅ */}
<Typography color="error.main">Could not save changes.</Typography>     {/* ❌ 3.64:1 fails */}
```

### Tinted alert / banner
```tsx
import { Alert, AlertTitle } from '@mui/material';

<Alert severity="success">
  <AlertTitle>Application submitted</AlertTitle>
  We'll email you when the review starts.
</Alert>
```

### Reading a color from the theme
```tsx
import { useTheme } from '@mui/material';

const theme = useTheme();
const accent = theme.palette.primary.main;   // → #196AE5 in light mode
```

## Do / Don't

| Do | Don't |
|---|---|
| `color="error.dark"` for inline error text | `color="error.main"` for inline error text |
| Use `<Alert severity="warning">` for amber notices | `color="warning.main"` on `<Typography>` (1.65:1 — fails) |
| Use `secondary` for decorative warmth | Use `secondary` to mean caution |
| Default to whitespace; sparse `<Divider>` | A `<Divider>` between every list item |
