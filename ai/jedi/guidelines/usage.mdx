> **Who this is for:** Anyone deciding whether a screen belongs in Jedi and how the screen should *behave* for power users.
> **What this covers:** The sovereign-posture rules — one primary action, flat outlined cards, scannable density, defaults that do the work.

## Why it matters

Jedi is the **sovereign** posture (Cooper): one codebase serves all internal/partner surfaces; users live in it for long stretches and become intermediate users fast. Optimize for the **returning** user — density is acceptable, keyboard navigation is expected, onboarding chrome is minimal. The design choices below all serve "you've been here before; let's get to work."

## Core principles

1. **One primary action per view.** Exactly one `contained` button; everything else `outlined` or `text`.
2. **Defaults do the work.** Pre-select the most common option; prefill from data the app already has.
3. **Forms: 7±2 visible fields per step (max 9).** Beyond that, split into steps with a visible progress indicator.
4. **Destructive actions confirm; reversible actions don't.** Delete/cancel get a confirm step naming the consequence; reversible actions execute immediately with undo.
5. **Empty states are starting points** — explain what belongs there and offer the action that fills it.
6. **Flat, outlined, no elevation.** Cards always `variant="outlined"` — no shadows. Shadows belong to true overlays (menus, dialogs, popovers).

## How to implement

### Flat outlined card
```tsx
import { Card, CardContent, Stack, Typography, Button } from '@mui/material';

<Card> {/* outlined by default via theme */}
  <CardContent>
    <Stack spacing={2}>
      <Typography variant="h5">Pending reviews</Typography>
      <Typography color="text.secondary">8 applications waiting on you.</Typography>
      <Button>Open queue</Button>   {/* single primary */}
    </Stack>
  </CardContent>
</Card>
```

### Don't raise cards
```tsx
<Card elevation={4}>...</Card>     {/* ❌ shadows belong to overlays */}
<Card>...</Card>                    {/* ✅ outlined */}
```

### Scannable table
```tsx
import { Table, TableHead, TableRow, TableCell } from '@mui/material';

<Table size="small">
  <TableHead>
    <TableRow>
      <TableCell>Cohort</TableCell>
      <TableCell align="right">Learners</TableCell>
      <TableCell align="right">Revenue</TableCell>
    </TableRow>
  </TableHead>
  {/* numeric columns right-aligned with tabular figures */}
</Table>
```

## Scannability rules

- Tables scan **down columns** — right-align numbers, left-align text, tabular figures.
- Front-load the distinguishing word ("Annual report 2024", not "Report (annual), 2024").
- Label and value visually distinct in key-value displays (label muted, value emphasized).
- One clear entry point per screen.
- Group by proximity, separate by whitespace — not dividers.

## Do / Don't

| Do | Don't |
|---|---|
| Use `variant="outlined"` on cards | Add `elevation={4}` for emphasis |
| Right-align numeric table columns | Left-align money columns |
| Confirm destructive actions (delete) | Confirm dialogs on save / edit |
| Build flows in steps with progress | One giant 25-field scroll-form |
